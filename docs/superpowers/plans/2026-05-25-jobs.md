# Jobs / Task Scheduler Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add run-to-completion jobs that run manually or on a cron schedule, each execution recorded as a JobRun with status + exit code, using the existing Linux runtime isolation.

**Architecture:** A `Job`/`JobRun` domain + SQLite tables (versioned migration). The runtime gains a `run_to_completion` path (userns/cgroup isolation, no ingress) that waits for exit. An in-process tokio scheduler evaluates persisted cron cursors (`next_run_at`/`last_enqueued_at`) and triggers background runs with a Forbid concurrency policy + retries. New `/v1/jobs` endpoints.

**Tech Stack:** Rust 2024, axum 0.8, rusqlite, tokio, serde, the `cron` crate, thiserror. Spec: `docs/superpowers/specs/2026-05-25-jobs.md`. **Depends on sub-project B (Projects).**

  - a failing run retries up to `max_retries` then `Failed`.
- [ ] **Step 2: Run** → FAIL.
- [ ] **Step 3: Implement** — `Scheduler { store, runtime, acquirer, command_runner }`; `tick(now)` claims all due jobs by advancing their persisted schedule cursor, then either records `Skipped` for active jobs or creates `Pending` runs and spawns execution. `execute_run(job, run)` materializes the rootfs bundle from `job.source`, marks `Running`, calls `run_to_completion`, maps exit 0 -> `Succeeded` else retry/`Failed`; retries update the same run's attempt count. ## Notes

- Jobs have no ingress: no socket/bridge/route/health — keep that path out of the
  run-to-completion code.
- Job `env` is non-secret only; keep credentials as `SecretRef`/SOPS references,
  never raw values in SQLite.
- Do not mutate shared rootfs bundle `process.json` per run; command/env overrides
  belong in the job runtime plan.
- Reconcile orphan `Pending`/`Running` runs on startup so a crash never leaves
  them stuck.
- Manual run returns 202 after enqueue; completion is observed through run history.
- Builds on B (project_id + versioned migrations); RBAC gates the endpoints later.
- Frontend is the companion plan `2026-05-25-jobs-frontend.md`.
