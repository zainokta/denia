# RBAC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add users with password login, session + API bearer tokens, and project-scoped roles (admin/operator/viewer) plus a global super-admin, authorizing every `/v1` route.

**Architecture:** New `User`/`Role`/`ProjectMembership`/`ApiToken`/`Session` domain types + SQLite tables (versioned migration). A `src/auth.rs` resolves a bearer to a `Principal` and provides `require_super_admin` / `require_project_role(min)` middleware that replaces the single-token guard. Project-scoped handlers resolve the target project (via path or the service's `project_id`) and compare the caller's membership role.

**Tech Stack:** Rust 2024, axum 0.8, rusqlite, serde, `argon2`, `rand`, `sha2`, thiserror. Spec: `docs/superpowers/specs/2026-05-25-rbac.md`. **Depends on sub-project B (Projects).**

- `docs/adr/008-rbac.md` + `docs/adr/README.md`, `AGENTS.md`.
- Tests colocated + `tests/backend_contract.rs`.
